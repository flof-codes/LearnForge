Hash Table Lookup Complexity

1. How Hashing Works

A hash function maps a key to an array index in constant time. The key is transformed into a fixed-size integer, then reduced to an index via modulo.

2. Average vs Worst Case

Average case is O(1) because a good hash function distributes keys uniformly. Worst case is $$O(n)$$ when all keys collide into the same bucket.

3. The Load Factor

Performance depends on the load factor:

$$\alpha = \frac{n}{m}$$

where $$n$$ = number of entries and $$m$$ = number of buckets. Python dicts resize when $$\alpha > \frac{2}{3}$$.
